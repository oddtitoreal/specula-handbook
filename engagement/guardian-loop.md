# Specula Guardian

> Fonte: `specula-bos/docs/06_system-lifecycle.md`

Presidio continuo per brand che hanno già attraversato il BOS.

| Campo | Contenuto |
|---|---|
| **Trigger** | Completamento del Full Journey, oppure Brand Radar che segnala drift per due cicli consecutivi |
| **Durata** | Annuale — con clausola di Re-Speculation se i segnali divergono |
| **Frequenza** | Ogni trimestre: revisione scenari, rilevazione drift, Guardian Report |
| **Attori** | Circolo di Sintesi interno + Brand Alchemist |
| **Output** | Guardian Report trimestrale · Trigger di Re-Speculation se necessario |

---

## Il Guardian Loop

```
Scan mensile (Circolo)
↓
Sensemaking trimestrale → Memory Audit → Guardian Report
↓
Se drift rilevato per 2 cicli consecutivi → Re-Speculation
↓
Re-Speculation: si rientra nel layer più vicino all'origine del problema
(non si ricomincia da zero)
```

## La Re-Speculation

La Re-Speculation è sempre una decisione umana. Il sistema la suggerisce — il Circolo la attiva. Non è un fallimento del BOS: è il suo funzionamento corretto. Un brand che attraversa una Re-Speculation ha un sistema che apprende.
